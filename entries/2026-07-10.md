## Interview Practice

### Scenario

**Question:** A user reports that a critical application on a Linux server is intermittently slow, especially during peak usage hours. They suspect a background task might be consuming excessive resources. As a Platform Engineer, describe how you would investigate this issue using only the Linux commands and concepts covered this week:

1.  **Identify currently running processes and their resource consumption.** What tools would you use, and what key information would you look for?
2.  **Determine if any tasks are scheduled to run automatically at regular intervals (e.g., system-wide or user-specific).** What command(s) would you use to examine these schedules?
3.  **If you identify a suspicious scheduled task (specifically a cron job), how would you temporarily disable it for the next hour to observe its impact, ensuring you can easily re-enable it afterwards without losing its original configuration?**

### Architecture

**Question:** You're tasked with designing a system on a Linux server to handle a daily batch job for report generation. This job is a CPU-intensive script that needs to run every day at 2:00 AM. However, before the main report generation starts, the script must confirm the availability of an external data source. This check might involve a brief waiting period if the data source isn't immediately ready. To ensure other critical services aren't impacted, the report generation script should run with a lower CPU priority. Finally, the operations team needs to be able to monitor the status and resource usage of this report generation process.

Architect a solution using only the fundamental Linux tools and concepts taught, such as cron, at, sleep, process prioritization (nice), and process monitoring (ps, top). Describe the sequence of operations and the specific tools you would employ to meet these requirements, justifying your choices for scheduling, conditional execution, process management, and observability.

### Coding

**Question:** You are a Platform Engineer responsible for automating daily operational tasks on a Linux server. You need to set up two critical tasks:

1.  A script named `generate_daily_report.sh` located at `/opt/scripts/` must run every day at 01:00 AM (01:00).
2.  Immediately after `generate_daily_report.sh` successfully completes, a second script, `archive_old_data.sh` (also in `/opt/scripts/`), needs to execute. However, there is a strict requirement for a 20-minute waiting period *after* the report generation finishes before the archiving process begins, to ensure all related processes have fully settled and resources are free.

Using only standard Linux command-line tools discussed in the course, provide the exact commands you would use to configure this automated scheduling.
