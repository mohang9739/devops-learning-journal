## Interview Practice

### Scenario

**Question:** You are on-call for an internal microservice, `inventory-sync`, which is critical for real-time stock updates across multiple platforms. Users are reporting that product stock is not reflecting correctly, and the `inventory-sync` service seems to be stuck or not processing updates. The service is configured to run as a systemd service, its application logs are expected in `/var/log/inventory-sync/`, and its configuration files in `/etc/inventory-sync/`. You SSH into the server hosting this service to investigate.

Describe your step-by-step diagnostic process, including the specific Linux commands you would use and your reasoning at each stage, to identify why the service is failing to process updates. Focus on using basic commands related to processes, file system navigation, and permissions.

### Architecture

**Question:** Your team manages a legacy Linux VM hosting several critical, independent microservices. Recently, one or more services have started exhibiting intermittent high CPU usage, impacting the overall stability and performance of the VM and other co-located services. Without introducing containerization (like Docker/Kubernetes), describe your approach to:

1.  **Diagnosis:** How would you identify which specific service or process is responsible for the CPU spikes using standard Linux commands, and how would you gather historical data on its resource consumption over time?
2.  **Mitigation Strategy (OS-level):** Propose and justify strategies using only core Linux features (e.g., process management, permissions, resource control) to isolate or limit the resource usage of a misbehaving service, preventing it from starving others. Discuss the architectural trade-offs of these OS-level mitigation strategies in terms of complexity, security, performance, and maintainability.

### Coding

**Question:** You're an SRE on-call, and `api.example.com` is reporting `HTTP 500` errors. Upon inspecting the `nginx` error logs (`/var/log/nginx/error.log`), you find recurring entries similar to:
`[error] 12345#12345: *123 open() "/var/www/api.example.com/cache/tmp_data_12345" failed (13: Permission denied)`

The `api.example.com` application is configured to run as the `webappuser` and uses `/var/www/api.example.com/cache` for temporary file storage.

Using only standard Linux command-line tools, provide the exact commands for the following steps:

1.  **Identify the User:** What command(s) would you use to verify which user the `nginx` worker processes are actually running as on the system? Assume `nginx` is already running.
2.  **Inspect Permissions:** How would you check the current ownership and permissions of the `/var/www/api.example.com/cache` directory?
3.  **Correct Permissions:** If you discover that `webappuser` does *not* have write access to `/var/www/api.example.com/cache` and its current owner is `root:root`, provide the exact command(s) to fix the permissions such that `webappuser` can create and modify files within this directory, while ensuring only `webappuser` and members of its primary group can write, and others can only read its contents. The directory should also be traversable by others.

### Scenario

**Question:** Your product team has just landed a critical feature requiring a new microservice to be deployed to production within two weeks. This involves provisioning entirely new infrastructure: a dedicated EKS namespace, an RDS PostgreSQL instance, S3 buckets, and specific network policies, all managed via Terraform and Kubernetes manifests. Your team is under pressure, and you're keen to leverage the latest AI tools to accelerate this.

Describe a scenario where you would strategically use tools like GitHub Copilot, Cursor, or a custom CLI agent (e.g., integrating with OpenAI APIs) to generate and implement the initial Terraform configurations and Kubernetes manifests. What specific steps would you take to leverage these tools effectively? What are the primary benefits you anticipate? More importantly, what are the significant risks (e.g., security vulnerabilities, cost inefficiencies, compliance deviations, correctness issues, maintainability challenges) you'd need to mitigate when using AI-generated infrastructure code in a production environment at a company like Razorpay or Swiggy? How would you validate the AI's output before deploying it, and what processes would you put in place to ensure ongoing reliability and adherence to organizational standards?

### Architecture

**Question:** Your rapidly scaling product-based company aims to significantly boost the development velocity of its infrastructure-as-code (Terraform, Kubernetes manifests) and automation scripts (Python, Bash) across multiple Platform Engineering and DevOps teams. The leadership is keen on leveraging AI-powered coding assistants (e.g., GitHub Copilot, Cursor, or even internal CLI agents) to achieve this.

As a lead Platform Engineer, how would you architect the secure and sustainable adoption and integration of such AI tools into your existing engineering ecosystem? Discuss the critical architectural considerations you would address, including but not limited to:

1.  **Data Privacy & Security**: How would you prevent sensitive IaC details (e.g., internal service names, non-public resource configurations, cloud account IDs) from potentially leaking or being used in public AI model training?
2.  **Integration & Workflow**: How would these tools seamlessly integrate with your existing IDEs, version control (Git), CI/CD pipelines, and code review processes without introducing friction?
3.  **Governance & Consistency**: What mechanisms would you put in place to ensure AI-generated code adheres to company-specific best practices, security policies, established module structures, and internal domain knowledge?
4.  **Developer Experience & Trust**: How would you ensure the tools genuinely accelerate development while maintaining developer trust and productivity, rather than creating 'black box' code that's hard to debug or maintain?
5.  **Cost & Scalability**: What are the considerations for licensing, potential usage costs, and ensuring the solution scales efficiently across a growing engineering team and diverse technology stack?

### Coding

**Question:** Your team has identified a recurring issue of orphaned EBS volumes accumulating in your AWS accounts, leading to unnecessary costs. Your task is to develop a script that identifies all currently unattached EBS volumes in a specified region (`us-east-1` for this exercise) and then generates the necessary `aws ec2 delete-volume` commands to clean them up.

The script should:
1.  **Identify:** List the Volume IDs of all EBS volumes in `us-east-1` that are in an `available` state (i.e., not attached to any EC2 instance).
2.  **Generate Commands:** For each identified unattached volume, output a separate `aws ec2 delete-volume --volume-id <VOLUME_ID>` command.
3.  **Output Format:** Print the list of identified unattached Volume IDs first, followed by the individual delete commands.

Write a shell script (Bash) or Python script to accomplish this. Assume `aws cli` is configured with appropriate permissions. You don't need to actually execute the delete commands, just print them to standard output.
